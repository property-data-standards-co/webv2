# GitHub Repositories

All PDTF 2.0 development happens in the open under the [property-data-standards-co](https://github.com/property-data-standards-co) GitHub organisation.

:::note[Repo rename]
The website repository was renamed from `webv2` to `web`. Update your bookmarks and remotes accordingly.
:::

## Repositories

| Repository | Description |
|---|---|
| [spec](https://github.com/property-data-standards-co/spec) | PDTF 2.0 specification suite — protocol specs and implementation specs |
| [core](https://github.com/property-data-standards-co/core) | `@pdtf/core` — signing, verification, DIDs, status lists, TIR client, CLI |
| [schemas](https://github.com/property-data-standards-co/schemas) | JSON Schemas for PDTF entities and credentials |
| [tir](https://github.com/property-data-standards-co/tir) | Trusted Issuer Registry — live registry data |
| [web](https://github.com/property-data-standards-co/web) | This website (Astro/Starlight) |

## Getting involved

- **Report issues** on the relevant repository
- **Propose changes** via pull requests
- **Discuss** in GitHub Discussions on the [spec](https://github.com/property-data-standards-co/spec/discussions) repo
- **Read the specs** — start with the [Architecture Overview](/specs/00-architecture-overview/)
