## 1. Overview

The TIR implementation is deliberately simple: a JSON file in a git repo, a CLI for managing it, and a CI pipeline that validates, signs, and publishes. No database, no admin UI, no API server.

**Components:**

| Component | What | Where |
|-----------|------|-------|
| `registry.json` | Source of truth | `property-data-standards-co/pdtf-tir` repo |
| `tir-schema.json` | JSON Schema for validation | Same repo, `schema/` |
| `@pdtf/tir-tools` | CLI for add/remove/validate/sign | npm package |
| CI pipeline | Validate → sign → publish on merge | GitHub Actions |
| Published artifact | Signed VC at static URL | `https://tir.propdata.org.uk/v1/registry` |
| Bundled snapshot | Build-time TIR for offline bootstrap | Shipped with `@pdtf/vc-validator` |

**What changes go through PRs, what runs in CI:**

```
Developer/Admin                    GitHub                        GCS/CDN
─────────────────────────────────────────────────────────────────────────
                                                                 
  tir add-issuer ...  ──────┐                                    
  tir validate        ──────┤                                    
  git commit + push   ──────┤                                    
  Open PR             ──────┼──▶  CI: validate schema            
                            │     CI: check DID format            
                            │     CI: check proxyFor refs         
                            │     CI: resolve active DIDs         
                            │                                     
  PR approved + merge  ─────┼──▶  CI: validate (again)           
                            │     CI: sign registry (KMS)  ──────▶ Upload signed VC
                            │     CI: invalidate CDN cache ──────▶ CDN refreshes
                            │     CI: tag release                 
```

### 2.1 Why `signed/registry-vc.json` in the repo?

The signed VC is committed back to the repo after each publish. This means:

- Anyone can verify the signed registry without fetching from the CDN
- The git history of `signed/registry-vc.json` is the complete audit trail of published versions
- Offline verification is possible (clone repo, check signature)
- The CDN is just a cache — the repo is always the fallback

## 3. `registry.json` Format

The format is defined in Sub-spec 04 §4. The impl adds one field:

```json
{
  "$schema": "./schema/tir-schema.json",
  "version": "1.0",
  "updated": "2026-03-25T03:00:00Z",
  "signingInfo": {
    "issuerDid": "did:web:moverly.com",
    "publishUrl": "https://tir.propdata.org.uk/v1/registry",
    "lastSignedAt": "2026-03-25T03:05:00Z",
    "lastSignedVersion": "v1.0.3"
  },
  "issuers": { "..." : "..." },
  "userAccountProviders": { "..." : "..." }
}
```

`signingInfo` is metadata about the publishing pipeline. It tells consumers:
- Who signs the published VC (`issuerDid`)
- Where to fetch the signed version (`publishUrl`)
- When it was last signed and what git tag it corresponds to

This field is updated by CI, not by humans.

### 4.3 `add-issuer` Command

Interactive and non-interactive modes for adding new issuers.

```typescript
// src/commands/add-issuer.ts

import { Command } from 'commander';
import { readFileSync, writeFileSync } from 'node:fs';
import type { TrustedIssuerRegistry, IssuerEntry } from '../types.js';

interface AddIssuerOptions {
  registry: string;
  slug: string;
  name: string;
  did: string;
  paths: string[];
  trustLevel: 'rootIssuer' | 'trustedProxy';
  proxyFor?: string;
  status: 'active' | 'planned';
  contact?: string;
  website?: string;
}

export function registerAddIssuer(program: Command): void {
  program
    .command('add-issuer')
    .description('Add a new issuer entry to the registry')
    .requiredOption('--slug <slug>', 'Issuer slug (lowercase-hyphenated)')
    .requiredOption('--name <name>', 'Human-readable name')
    .requiredOption('--did <did>', 'Issuer DID')
    .requiredOption('--paths <paths...>', 'Authorised entity:paths (space-separated)')
    .requiredOption('--trust-level <level>', 'Trust level: rootIssuer or trustedProxy')
    .option('--proxy-for <slug>', 'Root issuer slug (required for trustedProxy)')
    .option('--status <status>', 'Initial status', 'active')
    .option('--contact <email>', 'Contact email')
    .option('--website <url>', 'Website URL')
    .option('--registry <path>', 'Path to registry.json', './registry.json')
    .action(async (opts: AddIssuerOptions) => {
      // 1. Validate slug format
      if (!/^[a-z][a-z0-9-]*$/.test(opts.slug)) {
        console.error(`❌ Invalid slug "${opts.slug}" — must match ^[a-z][a-z0-9-]*$`);
        process.exit(1);
      }

      // 2. Validate paths format
      const pathPattern = /^[A-Z][a-zA-Z]*:\/[a-zA-Z0-9/*_-][a-zA-Z0-9/*._-]*$/;
      for (const path of opts.paths) {
        if (!pathPattern.test(path)) {
          console.error(`❌ Invalid path "${path}" — must match Entity:/json/path format`);
          process.exit(1);
        }
      }

      // 3. Validate proxyFor requirement
      if (opts.trustLevel === 'trustedProxy' && !opts.proxyFor) {
        console.error('❌ --proxy-for is required when --trust-level is trustedProxy');
        process.exit(1);
      }

      // 4. Load registry
      const raw = readFileSync(opts.registry, 'utf-8');
      const registry: TrustedIssuerRegistry = JSON.parse(raw);

      // 5. Check for duplicate slug
      if (registry.issuers[opts.slug]) {
        console.error(`❌ Slug "${opts.slug}" already exists in registry`);
        process.exit(1);
      }

      // 6. Check proxyFor target exists
      if (opts.proxyFor && !registry.issuers[opts.proxyFor]) {
        console.error(`❌ proxyFor target "${opts.proxyFor}" not found in registry`);
        console.error('   Add the root issuer entry first (even as status: planned)');
        process.exit(1);
      }

      // 7. Check for duplicate DID
      const existingDid = Object.entries(registry.issuers)
        .find(([, entry]) => entry.did === opts.did);
      if (existingDid) {
        console.error(`❌ DID "${opts.did}" already in use by issuer "${existingDid[0]}"`);
        process.exit(1);
      }

      // 8. Build entry
      const entry: IssuerEntry = {
        name: opts.name,
        did: opts.did,
        authorisedPaths: opts.paths,
        trustLevel: opts.trustLevel,
        ...(opts.proxyFor ? { proxyFor: opts.proxyFor } : {}),
        status: opts.status as 'active' | 'planned',
        validFrom: new Date().toISOString(),
        validUntil: null,
        ...(opts.contact ? { contact: opts.contact } : {}),
        ...(opts.website ? { website: opts.website } : {}),
      };

      // 9. Insert (sorted by slug for stable diffs)
      registry.issuers[opts.slug] = entry;
      const sortedIssuers: Record<string, IssuerEntry> = {};
      for (const key of Object.keys(registry.issuers).sort()) {
        sortedIssuers[key] = registry.issuers[key];
      }
      registry.issuers = sortedIssuers;

      // 10. Update timestamp
      registry.updated = new Date().toISOString();

      // 11. Write
      writeFileSync(opts.registry, JSON.stringify(registry, null, 2) + '\n');

      console.log(`✅ Added issuer "${opts.slug}" (${opts.trustLevel})`);
      console.log(`   DID: ${opts.did}`);
      console.log(`   Paths: ${opts.paths.join(', ')}`);
      console.log(`   Status: ${opts.status}`);
      if (opts.proxyFor) {
        console.log(`   Proxy for: ${opts.proxyFor}`);
      }
      console.log(`\n   Next: git add ${opts.registry} && git commit && git push`);
    });
}
```

### 4.4 `validate` Command

```typescript
// src/commands/validate.ts

import Ajv from 'ajv';
import addFormats from 'ajv-formats';
import { readFileSync } from 'node:fs';

export async function validateRegistry(registryPath: string, schemaPath: string): Promise<{
  valid: boolean;
  errors: string[];
  warnings: string[];
  stats: { issuers: number; providers: number; active: number; planned: number };
}> {
  const errors: string[] = [];
  const warnings: string[] = [];

  // 1. Load and parse
  let registry: any;
  try {
    const raw = readFileSync(registryPath, 'utf-8');
    registry = JSON.parse(raw);
  } catch (err) {
    return { valid: false, errors: [`Failed to parse ${registryPath}: ${(err as Error).message}`], warnings, stats: { issuers: 0, providers: 0, active: 0, planned: 0 } };
  }

  // 2. Schema validation
  const schema = JSON.parse(readFileSync(schemaPath, 'utf-8'));
  const ajv = new Ajv({ allErrors: true, strict: false });
  addFormats(ajv);
  const validate = ajv.compile(schema);
  const valid = validate(registry);

  if (!valid && validate.errors) {
    for (const err of validate.errors) {
      errors.push(`Schema: ${err.instancePath} ${err.message}`);
    }
  }

  // 3. Cross-reference checks
  const issuerSlugs = new Set(Object.keys(registry.issuers || {}));

  for (const [slug, entry] of Object.entries(registry.issuers || {}) as [string, any][]) {
    // proxyFor must reference an existing slug
    if (entry.proxyFor && !issuerSlugs.has(entry.proxyFor)) {
      errors.push(`Issuer "${slug}": proxyFor "${entry.proxyFor}" not found in registry`);
    }

    // Proxy can't proxy for itself
    if (entry.proxyFor === slug) {
      errors.push(`Issuer "${slug}": proxyFor references itself`);
    }

    // Proxy's proxyFor should be a rootIssuer (or planned rootIssuer)
    if (entry.proxyFor && registry.issuers[entry.proxyFor]) {
      const target = registry.issuers[entry.proxyFor];
      if (target.trustLevel !== 'rootIssuer') {
        warnings.push(`Issuer "${slug}": proxyFor "${entry.proxyFor}" is not a rootIssuer (it's ${target.trustLevel})`);
      }
    }

    // Deprecated entries should have validUntil
    if (entry.status === 'deprecated' && !entry.validUntil) {
      warnings.push(`Issuer "${slug}": status is deprecated but no validUntil set`);
    }

    // Active proxies should have their proxyFor target at least planned
    if (entry.status === 'active' && entry.proxyFor) {
      const target = registry.issuers[entry.proxyFor];
      if (target && target.status === 'revoked') {
        errors.push(`Issuer "${slug}": active proxy for revoked root issuer "${entry.proxyFor}"`);
      }
    }
  }

  // 4. DID uniqueness (across issuers and providers)
  const allDids = new Map<string, string>();
  for (const [slug, entry] of Object.entries(registry.issuers || {}) as [string, any][]) {
    if (allDids.has(entry.did)) {
      errors.push(`DID "${entry.did}" used by both "${allDids.get(entry.did)}" and "${slug}"`);
    }
    allDids.set(entry.did, slug);
  }
  for (const [slug, entry] of Object.entries(registry.userAccountProviders || {}) as [string, any][]) {
    if (allDids.has(entry.did)) {
      // Account providers can share DIDs with their issuer entries (e.g. moverly)
      // This is valid — same org, different roles
      // Only warn, don't error
      warnings.push(`DID "${entry.did}" used by provider "${slug}" and issuer "${allDids.get(entry.did)}" — verify this is intentional`);
    }
  }

  // 5. Stats
  const issuerEntries = Object.values(registry.issuers || {}) as any[];
  const providerEntries = Object.values(registry.userAccountProviders || {}) as any[];

  return {
    valid: errors.length === 0,
    errors,
    warnings,
    stats: {
      issuers: issuerEntries.length,
      providers: providerEntries.length,
      active: [...issuerEntries, ...providerEntries].filter(e => e.status === 'active').length,
      planned: [...issuerEntries, ...providerEntries].filter(e => e.status === 'planned').length,
    },
  };
}
```

### 4.5 `sign` Command

Signs `registry.json` into a Verifiable Credential using the platform KMS key.

```typescript
// src/commands/sign.ts

import { readFileSync, writeFileSync } from 'node:fs';
import { VcSigner, KmsClient, KeyMetadataStore } from '@pdtf/key-manager';
import { getFirestore } from 'firebase-admin/firestore';

interface SignOptions {
  registry: string;
  output: string;
  issuerDid: string;
}

export async function signRegistry(opts: SignOptions): Promise<void> {
  // 1. Load registry
  const registry = JSON.parse(readFileSync(opts.registry, 'utf-8'));

  // 2. Build the TIR VC
  const tirVc: Record<string, unknown> = {
    '@context': [
      'https://www.w3.org/ns/credentials/v2',
      'https://schema.pdtf.org/tir/v1',
    ],
    id: `https://tir.propdata.org.uk/v1/registry`,
    type: ['VerifiableCredential', 'TrustedIssuerRegistryCredential'],
    issuer: opts.issuerDid,
    validFrom: new Date().toISOString(),
    credentialSubject: {
      id: 'https://tir.propdata.org.uk/v1/registry',
      type: 'TrustedIssuerRegistry',
      version: registry.version,
      updated: registry.updated,
      issuers: registry.issuers,
      userAccountProviders: registry.userAccountProviders,
    },
  };

  // 3. Sign with platform key
  const db = getFirestore();
  const signer = new VcSigner(new KmsClient(), new KeyMetadataStore(db));

  const signedVc = await signer.sign(tirVc, {
    keyCategory: 'platform',
  });

  // 4. Write signed VC
  writeFileSync(opts.output, JSON.stringify(signedVc, null, 2) + '\n');

  console.log(`✅ Signed TIR as VC`);
  console.log(`   Issuer: ${opts.issuerDid}`);
  console.log(`   Output: ${opts.output}`);
  console.log(`   Issuers: ${Object.keys(registry.issuers).length}`);
  console.log(`   Providers: ${Object.keys(registry.userAccountProviders).length}`);
}
```

### 4.6 `verify` Command

Verifies a signed TIR VC. No KMS access needed — pure crypto verification.

```typescript
// src/commands/verify.ts

import { readFileSync } from 'node:fs';
import { VcVerifier } from '@pdtf/key-manager';

export async function verifySignedRegistry(vcPath: string): Promise<{
  valid: boolean;
  issuer: string;
  signedAt: string;
  registryVersion: string;
  issuers: number;
  providers: number;
}> {
  const vc = JSON.parse(readFileSync(vcPath, 'utf-8'));
  const verifier = new VcVerifier();

  const result = await verifier.verify(vc, {
    skipTirCheck: true,    // Can't TIR-check the TIR itself!
    skipRevocationCheck: true,
  });

  if (!result.valid) {
    throw new Error(`Signature verification failed: ${result.error}`);
  }

  const subject = vc.credentialSubject as Record<string, unknown>;

  return {
    valid: true,
    issuer: vc.issuer as string,
    signedAt: vc.proof?.created as string,
    registryVersion: subject.version as string,
    issuers: Object.keys(subject.issuers as object).length,
    providers: Object.keys(subject.userAccountProviders as object).length,
  };
}
```

### 4.7 `lookup` Command

```typescript
// src/commands/lookup.ts

export function lookupIssuer(
  registry: TrustedIssuerRegistry,
  query: string,
): LookupResult | null {
  // Try by slug first
  if (registry.issuers[query]) {
    return {
      section: 'issuers',
      slug: query,
      entry: registry.issuers[query],
    };
  }

  // Try by DID
  for (const [slug, entry] of Object.entries(registry.issuers)) {
    if (entry.did === query) {
      return { section: 'issuers', slug, entry };
    }
  }

  // Try account providers
  if (registry.userAccountProviders[query]) {
    return {
      section: 'userAccountProviders',
      slug: query,
      entry: registry.userAccountProviders[query],
    };
  }

  for (const [slug, entry] of Object.entries(registry.userAccountProviders)) {
    if (entry.did === query) {
      return { section: 'userAccountProviders', slug, entry };
    }
  }

  return null;
}
```

```typescript
// src/commands/update-status.ts

export function registerUpdateStatus(program: Command): void {
  program
    .command('update-status')
    .description('Change an entry\'s status')
    .requiredOption('--slug <slug>', 'Entry slug')

      if (!entry) {
        console.error(`❌ Slug "${opts.slug}" not found`);

      }

      const oldStatus = entry.status;
      entry.status = opts.status;

      if (opts.validUntil) {
        entry.validUntil = opts.validUntil;
      }

      // Auto-set validUntil for deprecation if not provided
      if (opts.status === 'deprecated' && !entry.validUntil && !opts.validUntil) {
        // Default: 6 months from now
        const sixMonths = new Date();
        sixMonths.setMonth(sixMonths.getMonth() + 6);
        entry.validUntil = sixMonths.toISOString();
        console.log(`   Auto-set validUntil to ${entry.validUntil} (6 months)`);
      }

      registry.updated = new Date().toISOString();

      writeFileSync(opts.registry, JSON.stringify(registry, null, 2) + '\n');
      console.log(`✅ "${opts.slug}": ${oldStatus} → ${opts.status}`);
    });
}
```

```typescript
// src/commands/diff.ts

export function diffRegistries(
  oldRegistry: TrustedIssuerRegistry,
  newRegistry: TrustedIssuerRegistry,
): RegistryDiff {
  const changes: RegistryChange[] = [];

  // Compare issuers
  const allSlugs = new Set([
    ...Object.keys(oldRegistry.issuers),
    ...Object.keys(newRegistry.issuers),
  ]);

  for (const slug of allSlugs) {
    const oldEntry = oldRegistry.issuers[slug];
    const newEntry = newRegistry.issuers[slug];

    if (!oldEntry && newEntry) {
      changes.push({ type: 'added', section: 'issuers', slug, entry: newEntry });
    } else if (oldEntry && !newEntry) {
      changes.push({ type: 'removed', section: 'issuers', slug, entry: oldEntry });
    } else if (oldEntry && newEntry) {
      // Check for field changes
      if (oldEntry.status !== newEntry.status) {
        changes.push({
          type: 'status-changed',
          section: 'issuers',
          slug,
          from: oldEntry.status,
          to: newEntry.status,
        });
      }
      if (JSON.stringify(oldEntry.authorisedPaths) !== JSON.stringify(newEntry.authorisedPaths)) {
        changes.push({
          type: 'paths-changed',
          section: 'issuers',
          slug,
          from: oldEntry.authorisedPaths,
          to: newEntry.authorisedPaths,
        });
      }
      if (oldEntry.did !== newEntry.did) {
        changes.push({
          type: 'did-changed',
          section: 'issuers',
          slug,
          from: oldEntry.did,
          to: newEntry.did,
        });
      }
    }
  }

  // Same for providers (similar logic, omitted for brevity)

  return { changes, timestamp: new Date().toISOString() };
}
```

## 5. Entity:Path Matcher

The path matching logic is shared between `@pdtf/tir-tools` (for validation) and `@pdtf/vc-validator` (for runtime verification). Published as part of `@pdtf/tir-tools` and re-exported by `@pdtf/vc-validator`.

```typescript
// src/path-match.ts

/**
 * Check whether a given entity:path is authorised by a list of authorised paths.
 *
 * Rules:
 * - Entity type must match exactly (case-sensitive)
 * - Path must match exactly, OR be covered by a wildcard
 * - Wildcard `/*` at end of a path covers all descendant paths
 * - `Entity:/*` covers all paths on that entity
 *
 * @example
 * isPathAuthorised('Property', '/energyEfficiency/certificate', [
 *   'Property:/energyEfficiency/*',
 * ]) // → true
 *
 * isPathAuthorised('Property', '/buildingConstruction/wallType', [
 *   'Property:/energyEfficiency/*',
 * ]) // → false
 */
export function isPathAuthorised(
  entityType: string,
  dataPath: string,
  authorisedPaths: string[],
): boolean {
  for (const authorisedPath of authorisedPaths) {
    const colonIndex = authorisedPath.indexOf(':');
    if (colonIndex === -1) continue;

    const authEntity = authorisedPath.slice(0, colonIndex);
    const authPath = authorisedPath.slice(colonIndex + 1);

    // Entity type must match exactly
    if (authEntity !== entityType) continue;

    // Exact match
    if (authPath === dataPath) return true;

    // Wildcard match
    if (authPath.endsWith('/*')) {
      const prefix = authPath.slice(0, -2); // Remove /*

      // "Entity:/*" matches everything on that entity
      if (prefix === '') return true;

      // "/foo/*" matches "/foo", "/foo/bar", "/foo/bar/baz"
      if (dataPath === prefix || dataPath.startsWith(prefix + '/')) {
        return true;
      }
    }
  }

  return false;
}

/**
 * Check all entity:paths in a credential against an issuer's authorised paths.
 * Returns the list of uncovered paths (empty if all are covered).
 */
export function checkCredentialPaths(
  credentialPaths: Array<{ entity: string; path: string }>,
  authorisedPaths: string[],
): { covered: boolean; uncoveredPaths: string[] } {
  const uncovered: string[] = [];

  for (const { entity, path } of credentialPaths) {
    if (!isPathAuthorised(entity, path, authorisedPaths)) {
      uncovered.push(`${entity}:${path}`);
    }
  }

  return {
    covered: uncovered.length === 0,
    uncoveredPaths: uncovered,
  };
}
```

### 6.2 Publish Pipeline

Runs on merge to `main` when `registry.json` has changed:

```yaml
# .github/workflows/publish.yml
name: Sign & Publish Registry

on:
  push:
    branches: [main]
    paths: ['registry.json']

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: write        # To commit signed/registry-vc.json
      id-token: write        # For Workload Identity Federation

    steps:
      - uses: actions/checkout@v4
        with:
          # Need to push the signed VC back
          token: ${{ secrets.GITHUB_TOKEN }}

      - uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Install dependencies
        run: npm ci

      - name: Validate (safety check)
        run: npx @pdtf/tir-tools validate --registry registry.json --schema schema/tir-schema.json

      # Authenticate to GCP via Workload Identity Federation
      # No service account key files — GitHub's OIDC token is exchanged for GCP credentials
      - id: auth
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ vars.GCP_WORKLOAD_IDENTITY_PROVIDER }}
          service_account: ${{ vars.GCP_TIR_PUBLISHER_SA }}

      - uses: google-github-actions/setup-gcloud@v2

      # Sign the registry with the platform KMS key
      - name: Sign registry
        run: |
          npx @pdtf/tir-tools sign \
            --registry registry.json \
            --output signed/registry-vc.json \
            --issuer-did did:web:moverly.com
        env:
          # The sign command uses @pdtf/key-manager which auto-discovers GCP credentials
          # from the Workload Identity Federation setup above
          GOOGLE_CLOUD_PROJECT: pdtf-platform-prod

      # Verify the signature we just created (sanity check)
      - name: Verify signature
        run: npx @pdtf/tir-tools verify --vc signed/registry-vc.json

      # Upload to GCS for CDN serving
      - name: Publish to GCS
        run: |
          gsutil -h "Content-Type:application/vc+ld+json" \
                 -h "Cache-Control:public, max-age=3600" \
                 cp signed/registry-vc.json \
                 gs://pdtf-tir-prod/v1/registry

      # Invalidate CDN cache
      - name: Invalidate CDN
        run: |
          gcloud compute url-maps invalidate-cdn-cache pdtf-tir-urlmap-prod \
            --path="/v1/registry" \
            --async
        continue-on-error: true  # CDN invalidation is best-effort

      # Commit the signed VC back to the repo
      - name: Commit signed VC
        run: |
          git config user.name "PDTF TIR Bot"
          git config user.email "tir-bot@moverly.com"
          git add signed/registry-vc.json
          git diff --cached --quiet || git commit -m "chore: update signed registry VC [skip ci]"
          git push

      # Tag the release
      - name: Create release tag
        run: |
          VERSION=$(jq -r '.version' registry.json)
          TIMESTAMP=$(date +%Y%m%d%H%M%S)
          TAG="v${VERSION}.${TIMESTAMP}"
          git tag -a "$TAG" -m "TIR publish: $TAG"
          git push origin "$TAG"
```

## 8. Verifier Integration

### 8.1 TIR Checker (for `@pdtf/vc-validator`)

```typescript
// In @pdtf/vc-validator package

import { VcVerifier } from '@pdtf/key-manager';
import { isPathAuthorised } from '@pdtf/tir-tools';

interface TirCheckerOptions {
  /** Primary fetch URL */
  tirUrl: string;
  /** Fallback: GitHub raw URL */
  fallbackUrl?: string;
  /** Bundled bootstrap TIR (from build time) */
  bootstrap?: Record<string, unknown>;
  /** Cache TTL in ms (default: 3600_000 = 1 hour) */
  cacheTtlMs?: number;
  /** Max stale cache age in ms (default: 86400_000 = 24 hours) */
  maxStaleCacheMs?: number;
  /** Verify the TIR VC signature? (default: true) */
  verifySignature?: boolean;
}

export class TirChecker {
  private cache: {
    registry: any;
    fetchedAt: number;
    etag?: string;
  } | null = null;

  private verifier = new VcVerifier();

  constructor(private options: TirCheckerOptions) {
    // Initialise with bootstrap if provided
    if (options.bootstrap) {
      this.cache = {
        registry: options.bootstrap.credentialSubject || options.bootstrap,
        fetchedAt: 0, // Force refresh on first real check
      };
    }
  }

  /**
   * Check whether an issuer DID is trusted for the given entity:paths.
   */
  async check(
    issuerDid: string,
    entityPaths: Array<{ entity: string; path: string }>,
  ): Promise<TirVerificationResult> {
    const registry = await this.getRegistry();
    if (!registry) {
      return {
        trusted: false,
        issuerSlug: null,
        trustLevel: null,
        status: 'unknown',
        pathsCovered: false,
        uncoveredPaths: entityPaths.map(p => `${p.entity}:${p.path}`),
        warnings: ['TIR unavailable'],
      };
    }

    // Find issuer by DID
    let foundSlug: string | null = null;
    let foundEntry: any = null;

    for (const [slug, entry] of Object.entries(registry.issuers || {})) {
      if ((entry as any).did === issuerDid) {
        foundSlug = slug;
        foundEntry = entry;
        break;
      }
    }

    // Also check account providers
    if (!foundEntry) {
      for (const [slug, entry] of Object.entries(registry.userAccountProviders || {})) {
        if ((entry as any).did === issuerDid) {
          foundSlug = slug;
          foundEntry = entry;
          break;
        }
      }
    }

    if (!foundEntry) {
      return {
        trusted: false,
        issuerSlug: null,
        trustLevel: null,
        status: 'unknown',
        pathsCovered: false,
        uncoveredPaths: entityPaths.map(p => `${p.entity}:${p.path}`),
        warnings: [`Issuer DID ${issuerDid} not found in TIR`],
      };
    }

    // Check status
    const warnings: string[] = [];
    if (foundEntry.status === 'revoked') {
      return {
        trusted: false,
        issuerSlug: foundSlug,
        trustLevel: foundEntry.trustLevel,
        status: 'revoked',
        pathsCovered: false,
        uncoveredPaths: [],
        warnings: [`Issuer ${foundSlug} has been revoked`],
      };
    }
    if (foundEntry.status === 'planned') {
      return {
        trusted: false,
        issuerSlug: foundSlug,
        trustLevel: foundEntry.trustLevel,
        status: 'planned',
        pathsCovered: false,
        uncoveredPaths: [],
        warnings: [`Issuer ${foundSlug} is planned but not yet active`],
      };
    }
    if (foundEntry.status === 'deprecated') {
      warnings.push(`Issuer ${foundSlug} is deprecated`);
      if (foundEntry.validUntil) {
        warnings.push(`Credentials will be rejected after ${foundEntry.validUntil}`);
      }
    }

    // Check validity period
    const now = new Date();
    if (foundEntry.validFrom && new Date(foundEntry.validFrom) > now) {
      return {
        trusted: false,
        issuerSlug: foundSlug,
        trustLevel: foundEntry.trustLevel,
        status: foundEntry.status,
        pathsCovered: false,
        uncoveredPaths: [],
        warnings: [`Issuer ${foundSlug} not yet valid (validFrom: ${foundEntry.validFrom})`],
      };
    }
    if (foundEntry.validUntil && new Date(foundEntry.validUntil) < now) {
      return {
        trusted: false,
        issuerSlug: foundSlug,
        trustLevel: foundEntry.trustLevel,
        status: foundEntry.status,
        pathsCovered: false,
        uncoveredPaths: [],
        warnings: [`Issuer ${foundSlug} has expired (validUntil: ${foundEntry.validUntil})`],
      };
    }

    // Check paths (only for issuer entries, not account providers)
    let pathsCovered = true;
    const uncoveredPaths: string[] = [];
    if (foundEntry.authorisedPaths) {
      for (const { entity, path } of entityPaths) {
        if (!isPathAuthorised(entity, path, foundEntry.authorisedPaths)) {
          pathsCovered = false;
          uncoveredPaths.push(`${entity}:${path}`);
        }
      }
    }

    return {
      trusted: pathsCovered,
      issuerSlug: foundSlug,
      trustLevel: foundEntry.trustLevel,
      status: foundEntry.status,
      pathsCovered,
      uncoveredPaths,
      warnings,
    };
  }

  /** Fetch and cache the TIR */
  private async getRegistry(): Promise<any | null> {
    const ttl = this.options.cacheTtlMs ?? 3600_000;
    const maxStale = this.options.maxStaleCacheMs ?? 86400_000;

    // Fresh cache
    if (this.cache && (Date.now() - this.cache.fetchedAt) < ttl) {
      return this.cache.registry;
    }

    // Try fetch
    try {
      const vc = await this.fetchTirVc(this.options.tirUrl);
      const registry = vc.credentialSubject;
      this.cache = { registry, fetchedAt: Date.now() };
      return registry;
    } catch {
      // Try fallback
      if (this.options.fallbackUrl) {
        try {
          const vc = await this.fetchTirVc(this.options.fallbackUrl);
          const registry = vc.credentialSubject;
          this.cache = { registry, fetchedAt: Date.now() };
          return registry;
        } catch { /* fall through */ }
      }

      // Use stale cache
      if (this.cache && (Date.now() - this.cache.fetchedAt) < maxStale) {
        return this.cache.registry;
      }

      return null;
    }
  }

  private async fetchTirVc(url: string): Promise<any> {
    const response = await fetch(url, {
      headers: { Accept: 'application/vc+ld+json, application/json' },
    });
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    const vc = await response.json();

    // Verify signature if configured
    if (this.options.verifySignature !== false) {
      const result = await this.verifier.verify(vc as Record<string, unknown>, {
        skipTirCheck: true,
        skipRevocationCheck: true,
      });
      if (!result.valid) {
        throw new Error(`TIR VC signature invalid: ${result.error}`);
      }
    }

    return vc;
  }
}
```

## 11. Open Questions

| # | Question | Status |
|---|----------|--------|
| TQ1 | Should the published URL be `tir.propdata.org.uk` or `tir.moverly.com`? | Leaning `propdata.org.uk` (neutral domain) |
| TQ2 | Should CI also publish the unsigned `registry.json` alongside the signed VC? | Probably yes — useful for tools that don't need signature verification |
| TQ3 | Should the `tir-tools` CLI support reading from a signed VC (not just raw JSON)? | Yes — `lookup` and `validate` should accept both formats |
| TQ4 | Do we need a staging TIR? | Leaning yes — separate `registry-staging.json` in the same repo |
| TQ5 | Should the signed TIR VC include a `credentialStatus` for revocation? | No — the TIR is self-revoking (entries have `status: revoked`). A revoked TIR VC would break all verification. |

## Appendix A: Decision Log

| ID | Decision | Rationale |
|----|----------|-----------|
| T1 | Git-hosted static file, no database | Changes are rare, auditability is critical, git gives both for free |
| T2 | Signed VC as published artifact | Verifiers can authenticate the TIR using the same trust infrastructure as credentials |
| T3 | Workload Identity Federation for CI signing | No service account keys in GitHub Secrets — GitHub OIDC token exchanged for short-lived GCP credentials |
| T4 | Signed VC committed back to repo | Git history = complete audit trail of published versions |
| T5 | GCS + CDN for public serving | Same infrastructure pattern as status lists and DID documents |
| T6 | CLI tools for management | No admin UI needed — changes are infrequent and should be deliberate |
